# 第一人称解谜游戏策划草案（优化版）

本项目基于"第一人称"和"解谜"元素进行游戏创作，根据专周项目要求，本项目为3D游戏，大部分模型需由组内成员自己动手建模完成。第一人称视角移动与交互通过Unity内创建脚本实现，采用模块化工作流确保开发效率。

## 三、游戏策划

### 1. 核心玩法设计

#### 关卡结构

进入游戏后，场景中有一栋二层房屋和周边的院子，形成完整的探索空间。玩家作为调查员需要在房屋内外进行探索并与物体产生交互。

#### 交互系统优化

- **可视化进度反馈**：设计"调查笔记"系统，每完成一个交互就在物品栏中解锁一条线索或打勾调查项
- **交互分类**：将可交互物体分为"线索物品"、"解谜道具"、"剧情触发器"三类
- **物品栏系统**：允许玩家收集重要物品并随时查看相关信息

#### 子关卡设计（电学解谜）

- **前置线索铺设**：在主场景中设置原子结构图、电路图碎片、电池等道具
- **触发机制**：与原子核模型摆件交互后，通过环境变化（如灯光闪烁、空间扭曲）进入以电为主题的解谜关卡
- **解谜流程**：
  1. 收集指定数量的电池为电子门锁通电
  2. 寻找电路连接线索
  3. 解出密码并通关
- **难度梯度**：从简单电路连接到复杂逻辑谜题逐步递进

### 2. 游戏背景故事

#### 故事背景

一个科学家在家离奇失踪，留下了一栋充满谜题的房子。玩家作为传奇调查员奉命前往调查科学家失踪的原因，但与传统设定不同，本作中调查员将拥有更具体的个人背景。

#### 主角设定优化

- **调查员背景**：主角曾经历过亲人离世的痛苦，这使得他对科学家的故事产生强烈共鸣
- **情感纽带**：通过调查过程中的回忆闪回，展现主角与科学家相似的命运轨迹

#### 核心剧情发展

1. **初始探索**：在房屋中发现各种科学实验痕迹和家庭照片
2. **关键发现**：完成对房屋的调查后发现隐藏房间
3. **剧情揭示**：
   - 房间内有科学家写的研究日志
   - 桌面上有年轻时的科学家和她妻子女儿的合照
   - 调查员注意到科学家此前确实是一个人居住，并没有妻子和女儿陪在身边
4. **时光机揭秘**：
   - 房间中有一个调查员从未见过的奇怪科学装置
   - 通过阅读研究日志了解到这是时光机
   - 科学家通过时光机穿越到过去，试图拯救在年轻时意外丧生的妻女

#### 科学家警告的强化

- **具体化后果**：科学家在日志中记录自己穿越后的失败经历
- **失败案例**：他成功回到过去，但无论怎么做都无法改变妻女的死亡
- **现实扭曲**：穿越行为导致现实世界出现不可逆扭曲（如邻居凭空消失、街道变化等）
- **核心警告**：不要随意研究和使用时光机，否则将影响现世历史或其它平行世界

#### 结局设计

1. **物品处理**：调查员将所有资料和与时光机有关的物品全部交给国家科学机构秘密保管
2. **开放性钩子**：
   - 调查员离开时收到一条来自未知号码的短信
   - 内容是一句只有科学家妻女才会说的话
   - 暗示时光机可能仍在运作，或者科学家的牺牲并未真正改变什么
3. **情感收尾**：墓园里多了一座没有尸体的坟墓，是那个科学家，旁边是他妻女的墓

### 3. 空间设计优化

#### 二层房屋空间利用

- **一楼**：日常活动区域，包含客厅、厨房、书房，主要用于基础线索收集
- **二楼**：私密空间，包含卧室、实验室，存放核心剧情物品
- **探索节奏**：通过楼梯、钥匙、密码锁等方式控制探索节奏
- **垂直动线**：设计需要上下楼的解谜环节，增强空间感

#### 院子功能

- 作为初始探索区域
- 隐藏外部线索（如被掩埋的实验记录）
- 提供环境氛围渲染

## 四、技术实现方案

### 1. 建模与资源工作流

- **C4D建模规范**：统一使用四边形布线，删除三角面和N-Gon
- **导出设置**：冻结变换，对齐轴心点，使用FBX格式导出
- **材质分离**：在C4D中只做白模，材质和风格化在Unity中完成

### 2. Unity开发规范

- **材质系统**：使用Standard Shader，支持PBR流程
- **交互系统**：基于Unity Event System实现物品交互
- **第一人称控制器**：使用Unity Character Controller组件

### 3. 性能优化策略

- **LOD系统**：为复杂模型设置多级细节
- **遮挡剔除**：合理设置Occlusion Culling
- **资源打包**：使用AssetBundle管理场景资源

## 七、总结

本优化版策划草案在保持原有核心创意的基础上，通过以下改进提升了项目的可执行性和游戏体验：

1. **强化了叙事衔接**：通过主角背景设定和科学家失败经历的具体化，增强了故事的情感张力
2. **优化了游戏机制**：增加了可视化进度反馈系统，改进了子关卡与主线的衔接方式
3. **完善了空间设计**：充分利用二层房屋结构，设计了更丰富的探索动线
4. **增加了开放结局**：通过悬念设置提升了游戏的回味价值
5. **强化了技术规范**：制定了详细的建模和开发规范，确保团队协作效率

该项目在专周时间内具备良好的可执行性，核心是把"调查进度可视化"和"子关卡与主线的叙事衔接"这两个关键点做扎实，相信能够产出令人满意的游戏作品。
